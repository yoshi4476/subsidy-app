# 補助金システム デプロイ・運用マニュアル (完全版: Railway & Vercel)

本ドキュメントは、この補助金システムをインターネット上に公開（デプロイ）し、SaaSとして安定運用していくための手順を初心者にも分かりやすく1から13のステップで解説した完全版マニュアルです。

---

## 🛠️ 1. 全体構成と使用ツール

本システムは、安全性と拡張性に優れた以下の3つのサービスを組み合わせて構築します。

| 役割                                            | 使用ツール (読み方)           | サービス内容とメリット                                                                                                                                    |
| :---------------------------------------------- | :---------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **ソースコード管理**                            | **GitHub**<br>(ギットハブ)    | 最新のソースコードを安全に保管する金庫です。コードを更新すると、後述の2つのプラットフォームが自動で検知し、数分で最新版が世の中に公開されます。           |
| **バックエンド (API)**<br>**データベース (DB)** | **Railway**<br>(レイルウェイ) | Python(FastAPI)の実行と、顧客データ(PostgreSQL)を保存する裏側の主力基盤です。使った分だけ（上限設定可能）の課金で、エンタープライズ級の安定性を誇ります。 |
| **フロントエンド (画面)**                       | **Vercel**<br>(ヴィセル)      | Next.js で作られた画面を爆速で世界中に配信する表示特化のプラットフォームです。この技術（Next.js）を作った大元の会社なので相性が完璧で、基本無料です。     |

---

## 🚀 2. デプロイ手順（1から13のステップ）

デプロイは大きく **「ソースコードの準備（フェーズ1）」「バックエンドの公開（フェーズ2）」「フロントエンドの公開（フェーズ3）」** に分かれます。

### 【フェーズ1：準備とソースコード管理】

#### ステップ 1: GitHub アカウントの作成と準備

1. [GitHub公式サイト](https://github.com/) にアクセスし、アカウントを作成します（無料）。
2. サインイン後、画面右上の「＋」ボタンから **「New repository」** をクリックします。
3. リポジトリ名（例：`hojokin-saas`）を入力し、企業・SaaS利用のため必ず **「Private（非公開）」** を選択して一番下の「Create repository」をクリックします。

#### ステップ 2: プロジェクトのコードを GitHub にアップロード

1. パソコンのターミナル（黒や青の画面）を開き、プロジェクトのフォルダ（`f:\補助金システム`）に移動します。
2. 以下のコマンドを上から1行ずつ順番に実行します。

   ```bash
   git init
   git add .
   git commit -m "初回のデプロイ準備"
   git branch -M main

   # 注意: 以下のURLはステップ1で作った、ご自身のGitHubのURLに書き換えてコピペしてください
   git remote add origin https://github.com/あなたのユーザー名/hojokin-saas.git

   git push -u origin main
   ```

3. GitHubの画面を更新して、自分のフォルダやファイルが表示されていればステップ2は成功です！

---

### 【フェーズ2：データベースとバックエンドの公開 (Railway)】

ここからはシステムの「顧客データ保存場所（DB）」と「頭脳処理（バックエンド）」をRailway上に作成します。

#### ステップ 3: Railway アカウントの作成と GitHub 連携

1. [Railway公式サイト](https://railway.app/) にアクセスし、「Login」から GitHub アカウントを使って登録・ログインします。
2. （初回のみ）利用規約に同意する画面などが出たら案内に従って進めてダッシュボード画面を開きます。

#### ステップ 4: データベース (PostgreSQL) の作成とURL取得

1. ダッシュボード画面の **「New Project」** をクリックします。
2. メニューから **「Provision PostgreSQL」** を選択します。
3. 数秒待つと、画面上に紫色の箱（PostgreSQL）が作成されますので、それをクリックします。
4. 右側に出てくるパネルの **「Variables」**（または Connect）タブを開きます。
5. `DATABASE_URL` という項目を探し、横にある目玉マークを押して表示された `postgresql://...` から始まる長い文字列をコピーし、メモ帳等に退避しておきます。

#### ステップ 5: バックエンド (FastAPI) の作成

1. 今作った紫の箱があるのと同じ画面（キャンバス）で、画面上部の **「+ New」** ボタンを押します。
2. メニューから **「GitHub Repo」** を選択し、ステップ2で作った `hojokin-saas` リポジトリを選択します。
3. （初回のみ「Configure GitHub App」のような承認ボタンが出たら許可してください）。
4. 画面にあなたのアプリの新しい箱が追加され、自動的に裏側で構築（ビルド）が始まります。

#### ステップ 6: 起動コマンドと環境変数の設定

追加したアプリアイコンをクリックし、右側のパネルで以下の2つの設定を行います。

**【6-A: Start Command (起動コマンド) の設定】**

1. 右側パネルの **「Settings」** タブを開き、下にスクロールして「Deploy」セクションを探します。
2. **「Start Command」** または「Custom Command」の欄に以下を正確に貼り付けて保存します。
   `uvicorn main:app --host 0.0.0.0 --port $PORT`

**【6-B: Variables (環境変数) の設定】**

1. 右側パネルの **「Variables」** タブを開きます。
2. **「New Variable」** を押し、以下の4つを追加（Add）します。

| VARIABLE_NAME (左側) | VALUE (右側に入れるデータ)                                                                           |
| :------------------- | :--------------------------------------------------------------------------------------------------- |
| `DATABASE_URL`       | ステップ4でコピーしておいた `postgresql://...` をそのまま貼り付けます。                              |
| `OPENAI_API_KEY`     | [OpenAI APIキー取得ページ](https://platform.openai.com/api-keys) で作成した `sk-proj-...` の文字列。 |
| `CORS_ORIGINS`       | `http://localhost:3000,http://localhost:3001` と入力します。                                         |
| `JGRANTS_API_KEY`    | JGrantsのAPIキー（なければ空欄でOKです）。                                                           |

#### ステップ 7: デプロイ実行の確認とURLの発行

1. 右側パネルの **「Deployments」** タブを開き、一番上の履歴が緑色の「Success」になるまで待ちます。
2. 成功したら、パネルの **「Settings」** タブに戻り、下にスクロールします。
3. **「Networking」** という項目にある **「Generate Domain」** ボタンを押します。
4. 自動的に `https://hojokin-saas-production....up.railway.app` という公開URLが作られます。
5. 作成されたURLをクリックして画面が開けば、バックエンドは大成功です！**このURLを退避（コピー）しておきましょう。**

#### ステップ 8: データベースのマイグレーション（テーブル作成）

データベースの「箱」の中に「テーブル（データの枠組み）」を作ります。

1. Railwayの **「Deployments」** タブにある **「Console」** (または terminalアイコン) をクリックします。
2. 以下を入力して実行（Enterキー）します。
   `alembic upgrade head` （または `python init_db.py`）
3. これでデータベースの準備も完了です。

---

### 【フェーズ3：フロントエンドのデプロイ (Vercel)】

最後に、ユーザーが操作する「画面」を公開し、先ほどのバックエンド（Railway）と繋ぎ合わせます。

#### ステップ 9: Vercel アカウントの作成とインポート

1. [Vercel公式サイト](https://vercel.com/) にアクセスし、「Sign Up」から GitHub アカウントを利用して登録・ログインします。
2. ダッシュボードの **「Add New...」** ボタンから **「Project」** を選択します。
3. 自分のリポジトリリストから `hojokin-saas` の横にある **「Import」** を押します。

#### ステップ 10: Vercel プロジェクトの設定

1. プロジェクト名（例：`hojokin-saas-web`）を設定し、Framework Preset は `Next.js` のままにします。
2. **Root Directory** の右側の「Edit」を押し、`frontend` フォルダを選択して保存します。

#### ステップ 11: Google Cloud Console でのログイン準備 (利用する場合)

（※独自ログインのみを使う場合は、このステップは飛ばしてステップ12へ進んでください）

1. [Google Cloud Console](https://console.cloud.google.com/) にログインします。
2. 「API とサービス」＞「認証情報」から「＋認証情報を作成」＞「OAuth クライアント ID」を選びます。
3. アプリケーションの種類: `ウェブ アプリケーション`
   承認済みのJavaScriptオリジン: `https://あなたのVercelのURL`
   承認済みのリダイレクトURI: `https://あなたのVercelのURL/api/auth/callback/google`
4. 作成を押して表示された **「クライアント ID」** と **「クライアント シークレット」** をメモします。

#### ステップ 12: フロントエンドの環境変数設定

Vercelの画面に戻り、**Environment Variables** のアコーディオンを開いて以下を入力し「Add」を押していきます。

**① バックエンド（Railway）との接続設定**
| Name (左側) | Value (右側に入れるデータ) |
| :--- | :--- |
| `NEXT_PUBLIC_API_URL` | ステップ7の最後で退避したRailwayのURLを貼り付けます。※最後の斜線 `/` は**絶対に入れない**。 |

**② Googleログインの設定**
| Name (左側) | Value (右側に入れるデータ) |
| :--- | :--- |
| `NEXTAUTH_URL` | Vercelでこれから割り当てられるWebサイトのURL（例：`https://hojokin-saas-web.vercel.app`） |
| `NEXTAUTH_SECRET` | セキュリティ用の適当なパスワード文字列（例：`ab12cd34ef56gh78ij90kl12mn34op56`） |
| `GOOGLE_CLIENT_ID` | ここにステップ11で取得した「クライアント ID」を貼り付けます。 |
| `GOOGLE_CLIENT_SECRET` | ここにステップ11で取得した「シークレット」を貼り付けます。 |

#### ステップ 13: デプロイの実行と最終確認

1. すべて入力したらページ最下部の **「Deploy」** ボタンを押します！
2. 少し待つと、画面に花吹雪（Congratulations!）が表示されます。
3. 「Continue to Dashboard」を押し、「Visit」ボタンをクリックして、実際にブラウザで自分のSaaSシステムにアクセスしてみましょう。
4. 画面が綺麗に表示され、ログインやデータの読み込みが通常通り動けば、すべての構築が完了です！

## 🔄 3. 日々の運用と自動アップデート

一度、上記の1〜13の手順で構築してしまえば、2回目以降の「機能追加」や「バグ修正」といった更新作業は驚くほど簡単になります。
パソコンでコードを修正して保存したら、ターミナルで以下の3行のコマンドを打つだけです。

```bash
git add .
git commit -m "〇〇の機能を修正して使いやすくした"
git push
```

これだけで、**GitHubへのアップロードをRailwayとVercelが自動的に検知し**、数分以内に最新版のSaaSシステムが世の中に自動再配布されます。
（手動で毎回RailwayやVercelのボタンを押す必要は一切ありません！）

システム構築、本当にお疲れ様でした！SaaS事業としての素晴らしい船出を応援しております！
