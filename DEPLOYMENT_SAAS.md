# 🔰 日本一わかりやすい！補助金システム Web公開マニュアル (Railway & Vercel版)

このマニュアルは、「プログラミングの難しいことはよくわからない」という方でも、順番通りにポチポチと進めるだけで、システムを世界中の人（顧客）に使ってもらえるようにインターネット上へ公開（デプロイ）できる手順書です。

---

## 🌟 1. 使う3つのサービスの役割（身近な例え）

システムを公開するために、今回は以下の3つの「超優秀な無料の引っ越し業者」を使います。まずはこの3つに登録して繋ぎ合わせるイメージです！

| 使うサービスの名前             | 身近なもので例えると？            | なぜ使うの？（初心者向け解説）                                                                                                   |
| :----------------------------- | :-------------------------------- | :------------------------------------------------------------------------------------------------------------------------------- |
| **GitHub** <br>(ギットハブ)    | 🔒 **「絶対に燃えない金庫」**     | あなたのパソコンにあるプログラム一式を、インターネット上の安全な場所に保管するための金庫です。                                   |
| **Railway** <br>(レイルウェイ) | 🧠 **「裏で働くエリート社員」**   | 「頭脳（データを処理する）」と「記憶（顧客データを覚える）」を担当する裏方のサーバーです。Renderの進化版で、設定が一番簡単です。 |
| **Vercel** <br>(ヴィセル)      | 📺 **「超速いショーウィンドウ」** | お客さんが実際に見る「画面（フロントエンド）」を綺麗に、そして一瞬で表示してくれる超高性能なショーウィンドウです。               |

## 🏃‍♂️ 2. 公開までの全手順（順番にやってみましょう！）

### 【第1章】ソースコードを金庫（GitHub）に預ける

#### ステップ 1: GitHub アカウントの作成と準備

1. [GitHub公式サイト](https://github.com/) にアクセスし、アカウントを作成します（無料）。
2. サインイン後、画面右上の「＋」ボタンから **「New repository」** をクリックします。
3. リポジトリ名（例：`subsidy-app`）を入力し、企業・SaaS利用のため必ず **「Private（非公開）」** を選択して一番下の「Create repository」をクリックします。

#### ステップ 2: プロジェクトのコードを GitHub にアップロード

1. パソコンのターミナル（黒や青の画面）を開き、プロジェクトのフォルダ（`f:\補助金システム`）に移動します。
2. 以下のコマンドを上から1行ずつ順番に実行します。

   ```bash
   git init
   git add .
   git commit -m "初回のデプロイ準備"
   git branch -M main

   # 注意: 以下のURLはステップ1で作った、ご自身のGitHubのURLに書き換えてコピペしてください
   git remote add origin https://github.com/yoshi4476/subsidy-app.git

   git push -u origin main
   ```

3. GitHubの画面を更新して、自分のフォルダやファイルが表示されていればステップ2は成功です！

---

### 【第2章】裏側の頭脳（Railway）を設定する

ここでは、システムの裏側（データ処理や保存）をする優秀な社員（Railway）を雇います。

#### ステップ 3: Railwayに登録する

1. [Railway公式サイト](https://railway.app/) を開きます。
2. 「Login」ボタンから進み、「GitHubアカウントでログイン」を選びます。同意画面が出たらOKを押して進みます。

#### ステップ 4: 顧客データを入れる「引き出し」を作る

1. Railwayの画面で **「New Project」** を押します。
2. 出てきたメニューから **「Provision PostgreSQL」**（データベースのこと）を選びます。
3. 画面上に紫色の四角い箱（引き出し）ができるので、それをクリックします。
4. 右側に出るメニューから **「Variables」**（変数）タブを開きます。
5. `DATABASE_URL` という項目を見つけ、目玉マークを押して出てきた `postgresql://` から始まる長い文字をコピーして、メモ帳に貼り付けておきます。（後で使います！）

#### ステップ 5: バックエンド (FastAPI) の作成

1. 今作った紫の箱があるのと同じ画面（キャンバス）で、画面上部の **「+ New」** ボタンを押します。
2. メニューから **「GitHub Repo」** を選択し、ステップ2で作った `subsidy-app` リポジトリを選択します。
3. （初回のみ「Configure GitHub App」のような承認ボタンが出たら許可してください）。
4. 画面にあなたのアプリの新しい箱が追加され、自動的に裏側で構築（ビルド）が始まります。！

#### ステップ 6: 頭脳に「パスワード」と「設計図」を教える

追加された箱（あなたのアプリ）をクリックし、右側のパネルで以下の2つを設定します。

**【① 動かし方（Start Command）を教える】**

1. 右パネルの **「Settings」** タブを開き、下にスクロールして「Deploy」という場所を見つけます。
2. **「Custom Command」**(または Start Command) という欄に、以下をピッタリ貼り付けて保存します。
   `uvicorn main:app --host 0.0.0.0 --port $PORT`

**【② 使うパスワード（Variables）を教える】**

1. 右パネルの **「Variables」** タブを開きます。
2. **「New Variable」** を押し、以下の4つを1つずつ「追加（Add）」して教えてあげます。

| 入れる名前 (左側の枠) | 教える内容 (右側の枠)                                             |
| :-------------------- | :---------------------------------------------------------------- |
| `DATABASE_URL`        | 先ほどステップ4でメモ帳に貼り付けた `postgresql://...` の文字     |
| `OPENAI_API_KEY`      | OpenAIから取得した `sk-...` の文字（AIを動かすためのパスワード）  |
| `CORS_ORIGINS`        | ひとまず `http://localhost:3000,http://localhost:3001` と入れます |
| `JGRANTS_API_KEY`     | JGrantsのAPIキー（なければ空欄でOKです）                          |

#### ステップ 7: デプロイ（構築）の完了とURLのゲット

1. 右パネルの **「Deployments」** タブを開き、一番上の履歴が緑色の「Success」になるまで1〜2分待ちます。
2. 成功したら、もう一度 **「Settings」** タブに戻って下にスクロールします。
3. **「Networking」** のところにある **「Generate Domain」** ボタンを押します。
4. `https://hojokin...up.railway.app` という特別なURLが生まれます！
   （これが裏側用のURLです。**メモ帳にコピーしておいてください！**）

#### ステップ 8: データベースのセットアップ仕上げ

最後に、先ほど作った「引き出し（DB）」の中に、ユーザーデータや補助金データを入れるための「仕切り（テーブル）」を自動で作らせます。

1. Railwayの **「Deployments」** タブの下にある **「Console」**（黒い枠のアイコン）を押します。
2. そこに以下を半角で入力してEnterを押します。
   👉 **`python ensure_tables.py`**
3. これで裏側の作業はすべてコンプリートです！ 🎉

### 【第3章】お客さんの見る画面（Vercel）の設定

最後に、ユーザーがスマホやパソコンで見る「きれいな画面」を作り、先ほどのRailwayの頭脳と合体させます。

#### ステップ 9: Vercel アカウントの作成とインポート

1. [Vercel公式サイト](https://vercel.com/) にアクセスし、「Sign Up」から GitHub アカウントを利用して登録・ログインします。
2. ダッシュボードの **「Add New...」** ボタンから **「Project」** を選択します。
3. 自分のリポジトリリストから `subsidy-app` の横にある **「Import」** を押します。

#### ステップ 10: Vercelの基本設定

1. プロジェクト名（URLの一部になります。例：`subsidy-app`など）を決めます。
2. **Root Directory** の横にある「Edit」ボタンを押し、`frontend` というフォルダを選んで保存します。

#### ステップ 11: Googleログインの設定（スキップも可）

Googleアカウントでシステムにログインできるようにするための設定です。

1. [Google Cloud Console](https://console.cloud.google.com/) にログインします。
2. 「API とサービス」＞「認証情報」＞「＋認証情報を作成」＞「OAuth クライアント ID」の順に進みます。
3. 種類: `ウェブ アプリケーション`
   承認済みのJavaScriptオリジン: `https://あなたのVercelのURL`
   承認済みのリダイレクトURI: `https://あなたのVercelのURL/api/auth/callback/google`
   と入力して作成します。
4. 出てきた **「クライアント ID」** と **「クライアント シークレット」** をメモ帳にコピーします。

#### ステップ 12: Vercelに「パスワード」を教える

Vercelの画面に戻り、**Environment Variables（環境変数）** という項目を開いて、以下を追加（Add）していきます。

| 入れる名前 (左側の枠)  | 教える内容 (右側の枠)                                                                                         |
| :--------------------- | :------------------------------------------------------------------------------------------------------------ |
| `NEXT_PUBLIC_API_URL`  | ステップ7でメモした RailwayのURL (`https://....railway.app`) <br>※一番最後の`/`は**絶対に入れないでください** |
| `NEXTAUTH_URL`         | VercelのサイトURL（例：`https://subsidy-app.vercel.app`）                                                     |
| `NEXTAUTH_SECRET`      | 適当な英数字30文字以上（例：`abcd1234wxyz5678abcd1234wxyz5678`）                                              |
| `GOOGLE_CLIENT_ID`     | 先ほどの「クライアント ID」を貼り付けます                                                                     |
| `GOOGLE_CLIENT_SECRET` | 先ほどの「クライアント シークレット」を貼り付けます                                                           |

#### ステップ 13: いざ公開！🚀

1. すべて入力したら、一番下にある青い **「Deploy」** ボタンを押します！
2. 2〜3分待つと、ドドン！と花吹雪（Congratulations!）が舞う画面になります。
3. 画面の中の「Visit」ボタンを押すと……
   **インターネット上に公開された、あなたの大切なシステムが開きます！**
4. ログインや操作ができれば、大成功です！
